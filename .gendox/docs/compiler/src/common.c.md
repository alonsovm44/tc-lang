# tc - Type Checking and Error Reporting Library Documentation

## Purpose:

This library is designed to provide a comprehensive error handling and type checking functionality. It includes functions for memory management (`xmalloc`, `xrealloc`, `xstrdup`), string manipulation (`str_push`, `str_add`, `str_printf`), and error reporting mechanisms like the `die` function.

### Key Features:
- **Error Handling:** The library provides a robust `die` function to terminate processes cleanly with an error message.
- **Memory Management:** Functions for memory allocation (`xmalloc`) and reallocation (`xrealloc`) are provided, along with a string duplication function (`xstrdup`).
- **String Manipulation:** The `str_push`, `str_add`, and `str_printf` functions facilitate efficient handling of strings in various operations.
- **Error Reporting:** Detailed error messages can be generated by leveraging the `tc_error` function, which includes line numbers and spans for pinpointing errors within source code.

## Usage:

### Initialization and Configuration:
```c
#include "common.h"
```

- Initialize a context structure to store source information using `tc_set_source`.
- This setup is necessary if you plan on including source filename and content into error messages.
  
```c
// Example initialization of the source context
const char *filename = "example.c";
const char *source = "int example() { return 0; }";
tc_set_source(filename, source);
```

### Memory Management:
- Allocate memory safely using `xmalloc` and `xrealloc`.
- Duplicate strings with caution using `xstrdup`.

```c
// Example of safe string duplication
char *safe_string = xstrdup("Safe String");
```

### String Manipulation:
Use the functions in `tc.c` for adding characters, appending content, or formatting output.

```c
Str s;
while (*text) str_push(&s, *text++);
str_printf(&s, "%s %d\n", "Hello ", 10);
printf("%s\n", s.data); // Output: Hello 10
```

### Error Handling:
For error reporting and logging, `tc_error` function is used. It supports detailed line numbers, spans of interest within lines, and formatting messages.

```c
// Reporting an error with a specific message and span
char *message = "An unexpected issue";
const char *ecode = "ERR_UNEXPECTED";
tc_error(ecode, 5, 7, 3, "%s", message);
```

## Behavior:

### Error Handling Function (`die`):
- Terminates the process with an error code if called.
- Utilizes `vfprintf(stderr, fmt, ap)` to output formatted strings followed by a newline and then calls `exit(1)`.

### Memory Functions:
- `xmalloc(size_t size):` allocates memory. If allocation fails, it triggers an exit due to out of memory condition.
- `xrealloc(void *ptr, size_t size):` reallocates memory for the given pointer. It also includes a boundary check and allocation handling.
- `xstrdup(const char *s):` duplicates strings, ensuring proper copying including null terminator.

### String Manipulation Functions:
- **str_push(Str *s, char c):** Adds characters to string structures (`Str`). Resizes array as necessary due to buffer expansion rules.
- **str_add(Str *s, const char *text):** Appends text by sequentially pushing characters from the source into the destination.
- **str_printf(Str *s, const char *fmt, ...):** Uses `vsnprintf` for formatting and appending content efficiently.

### Error Reporting Function (`tc_error`):
- Accepts an error code, line number, column number, span range, formatted message, and a variable argument list.
- Outputs the error using `fprintf(stderr, "%s%serror[%s]%s%s: ", RED, BOLD, ecode, RESET, BOLD);`.
- Optionally prints source file information if initialized with `tc_set_source`.
- Outputs line-based information such as line numbers, spans, and highlighted spans for improved debugging.
- Finally calls `exit(1)` to terminate the program.

These functions are designed to be modular and reusable throughout your codebase. Ensure to properly initialize error reporting structures or utilize them within a context where errors may arise.
